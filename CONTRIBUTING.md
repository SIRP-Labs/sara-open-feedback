# Contributing to SARA Open

Thank you for your interest in improving SARA Open!

## How to Contribute

Since SARA Open's source code is maintained in a private repository, community contributions happen through this feedback repo:

### Report Bugs
- Use the [Bug Report](https://github.com/SIRP-Labs/sara-open-feedback/issues/new?template=bug_report.yml) template
- Include steps to reproduce, expected vs actual behavior, and screenshots if possible

### Report Inaccurate Responses
- Use the [Inaccurate Response](https://github.com/SIRP-Labs/sara-open-feedback/issues/new?template=inaccurate_response.yml) template
- Include the exact query, SARA's response, and the correct information with sources

### Suggest Features
- Start a [Feature Request discussion](https://github.com/SIRP-Labs/sara-open-feedback/discussions/categories/feature-requests)
- Describe the problem you're solving and your proposed solution
- Upvote existing ideas you'd like to see built

### Ask Questions
- Use [Q&A discussions](https://github.com/SIRP-Labs/sara-open-feedback/discussions/categories/q-a) for usage questions
- Search existing discussions first — your question may already be answered

### Share Knowledge
- Share interesting use cases, workflows, or tips in [Show & Tell](https://github.com/SIRP-Labs/sara-open-feedback/discussions/categories/show-and-tell)

## Code Contributions

If you're interested in contributing code directly to SARA Open, please email [support@sirp.io](mailto:support@sirp.io) with:
- Your background and area of interest
- What you'd like to work on
- Your GitHub username

We review contributor requests and grant access on a case-by-case basis.

## Security Vulnerabilities

Please report security vulnerabilities privately to [security@sirp.io](mailto:security@sirp.io). Do **not** open public issues for security concerns.
