## What is SARA Open?

SARA (Security Analyst & Response Assistant) Open is a **free AI-powered cybersecurity analyst** built by [SIRP](https://sirp.io). It provides enterprise-grade security capabilities to analysts, researchers, and SOC teams — at no cost.

### Key Capabilities

- **Threat Intelligence** — CVE lookups, IP/domain/hash reputation, EPSS scores, MITRE ATT&CK mapping
- **Incident Analysis** — Triage, classification, severity assessment, and remediation guidance
- **Detection Engineering** — Sigma rule generation, KQL queries, YARA rules
- **Knowledge Base** — Continuously updated with CVEs, KEV, EPSS, Sigma rules, and GCC regulatory frameworks (SAMA CSF, NCA ECC, PDPL)
- **OmniSense Integration** — Connect to SIRP's SOAR platform for live incident data, playbook execution, and SOC metrics
- **Web Research** — Real-time web search for emerging threats and zero-days

## About This Repository

This repository is the **community hub** for SARA Open. Use it to:

- **Report bugs** — Found something broken? [Open an issue](https://github.com/SIRP-Labs/sara-open-feedback/issues/new?template=bug_report.yml)
- **Request features** — Have an idea? [Start a discussion](https://github.com/SIRP-Labs/sara-open-feedback/discussions/categories/feature-requests)
- **Ask questions** — Need help? [Ask the community](https://github.com/SIRP-Labs/sara-open-feedback/discussions/categories/q-a)
- **Share use cases** — Show how you use SARA in your workflow

### Source Code

The SARA Open codebase is maintained in a **private repository** with controlled contributor access. This ensures code quality, security review, and responsible development of security tooling. If you're interested in contributing, please reach out to [support@sirp.io](mailto:support@sirp.io).

## Getting Started

1. Visit [sara-open.sirp.io](https://sara-open.sirp.io)
2. Sign up with your email
3. Start asking security questions — no setup required

**Pro users** can connect their OmniSense instance under Settings > OmniSense to unlock live incident data and playbook execution.
